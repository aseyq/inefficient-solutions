# The Emergence and Cultural Persistence of Ineffective Solutions

This repository contains the code and data necessary to reproduce analyses and figures in the paper _"The Emergence and Cultural Persistence of Ineffective Solutions."_ (Saral, Singh, Jacquet, Jang & Derex, 202X). 

## Repository Structure
| Path | Description |
|---|---|
| `code/` | Analysis scripts for main and supplementary figures/tables. |
| `code/fig2.R`, `code/fig3.R`, `code/fig4.R`, `code/fig5_data.R` | Main figure generation and supporting data prep. |
| `code/figA1.R` ... `code/figA10.R` | Supplementary figure scripts. |
| `code/tableA1.R`, `code/tableA2.R`, `code/tableA4.R` | Supplementary table scripts. |
| `code/zzz_optional_classify_gpt5.py` | Classifies advice into categories using GPT (requires `OPENAI_API_KEY`). |
| `code/zzz_optional_compare_manual_and_gpt_coding.R` | Compares manual coding with GPT coding outputs. |
| `code/zzz_optional_figA7_calclulate_semantic.py` | Creates semantic similarity inputs in `data/processed/` for `code/figA7.R`. |
| `data/` | Raw and derived datasets used by analyses. |
| `data/df_advice_gpt5.csv` | GPT-based coding output for advice messages. |
| `data/df_advice_manual_coding.csv` | Manual coding labels for advice messages. |
| `data/df_long.csv` | Main dataset used across analyses. |
| `data/gpt_template.csv` | Input template used by `code/zzz_optional_classify_gpt5.py`. |
| `data/processed/` | Processed outputs (including semantic similarity files). |
| `figures/` | Output figures generated by scripts. |
| `data_dictionary.md` | Variable descriptions for the datasets. |
| `requirements.txt` | Python dependencies for semantic processing and GPT classification scripts. |
| `run_all.R` | Runs all R scripts in `code/` and writes outputs to `figures/`. |

## Requirements

- R packages:
   - `tidyverse` (2.0.0)
   - `lme4` (1.1.36)
   - `lmerTest` (3.1.3)
   - `sjPlot` (2.9.0)
   - `purrr` (1.0.2)

- Python packages (see `requirements.txt`):
  All python code and dependencies are optional and only required for semantic similarity analyses (Fig A7) and GPT-based coding of advice (Not present in the manuscript, for comparison). The results of these analyses are included in the repository. 
   - `numpy` (2.4.2)
   - `pandas` (3.0.1) 
   - `openai` (2.21.0)
   - `sentence-transformers` (5.2.3)


You can install the required packages with the command:

```r
install.packages(c("tidyverse", "lme4", "lmerTest", "sjPlot", "purrr"))
```

Install Python dependencies with:

```bash
pip install -r requirements.txt
```

Notes for the semantic script (optional):
- `code/zzz_optional_figA7_calclulate_semantic.py` uses the public model `all-MiniLM-L6-v2`.
- No Hugging Face account is required.
- Requires download of the model the first time, which may take a few minutes.

## Usage

1. **Clone the repository**:
   ```bash
   git clone https://github.com/aseyq/inefficient-solutions.git
   ```

2. **Install R packages**:

   ```r
   install.packages(c("tidyverse", "lme4", "lmerTest", "sjPlot", "purrr"))
   ```

3. **Run analyses and generate figures**:

   Run all R scripts in `code/`:

   ```bash
   Rscript run_all.R
   ```

   Or run specific scripts manually:

   ```r
   source("code/fig2.R")
   source("code/fig3.R")
   source("code/fig4.R")
   source("code/figA7.R")
   ```

4. Output files (figures, tables, and intermediate products) are written to `figures/`.



**(optional - To regenerate semantic similarity files for Figure A7).**


1. **Install Python dependencies and run semantic preprocessing script** :

   ```bash
   pip install -r requirements.txt
   ```

   Then run:

   ```bash
   python code/zzz_optional_figA7_calclulate_semantic.py
   ```

   This recreates files in `data/processed/` that are required by `code/figA7.R`.

**(optional - To regenerate GPT-based coding of advice and comparison with manual coding).**
Please note that for this replication, we are including the GPT-based coding of advice messages for comparison with manual coding, and this will cost some API credits. Please make sure that you are aware of the costs associated with using the OpenAI API before running this part of the code.

1. **Set up OpenAI API key**:

   Ensure you have an OpenAI API key and set it as an environment variable 
   
   For macOS/Linux:

   ```bash
   export OPENAI_API_KEY="your_api_key_here"
   ```

   For Windows (PowerShell):

   ```powershell
   $env:OPENAI_API_KEY="your_api_key_here"
   ```

2. **Run GPT classification script**:

   ```bash
   python code/zzz_optional_classify_gpt5.py
   ```

   This generates `data/df_advice_gpt5.csv` with GPT-based coding of advice messages. This replication is not present in the manuscript but is included for comparison with manual coding.
